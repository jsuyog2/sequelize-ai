Natural Language → Sequelize

Query your DB
in plain English

sequelize-ai translates natural language into Sequelize queries, executes them in a secure isolated sandbox, and returns structured JSON — no SQL required.

↓ Install on npm ★ GitHub
quickstart.js 
const SequelizeAI = require("sequelize-ai");

const ai = new SequelizeAI(sequelize, {
  provider: "groq",
  apiKey: process.env.GROQ_API_KEY,
});

// Plain English → Sequelize → JSON
await ai.ask("get all products where stock is greater than 10");
await ai.ask("show top 5 most expensive Electronics products");
await ai.ask("total stock left and earnings if all products sold");
await ai.ask("what is the cheapest and most expensive product price");
Why sequelize-ai

Everything you need,
nothing you don't

🗣️

Natural Language

Ask questions in plain English. No need to know Sequelize syntax or SQL — just describe what you want.

🔒

Secure Sandbox

All generated code runs inside an isolated V8 context via isolated-vm — completely separate from your Node.js process.

🚫

Read-Only by Design

Only SELECT-equivalent methods are permitted. Write operations like destroy, update, and create are blocked.

💡

AI Column Hints

Add aiDescription to your model columns for richer LLM context and more accurate query generation.

🧮

Computed Columns

Derive values like stock * price or discounted prices on the fly — no raw SQL expressions needed.

⚡

Multi-Provider

Works with OpenAI, Gemini, Claude, Groq, and DeepSeek. Switch providers with a single config change.

🔀

Multi-Query

Ask compound questions that require multiple database calls. Results returned as a structured array.

📦

Structured Output

Every response returns a consistent { model, method, data } envelope — easy to consume in any app.

v1.1.0 Feature

AI Column Hints

Add aiDescription to your Sequelize model columns to give the LLM richer context. This leads to more accurate queries, especially for computed columns and ambiguous field names. It is never written to your database.

Without aiDescription less accurate 
const Product = sequelize.define("Product", {
  name:     { type: DataTypes.STRING },
  price:    { type: DataTypes.DECIMAL },
  category: { type: DataTypes.STRING },
  stock:    { type: DataTypes.INTEGER },
});

// LLM sees:
// name:STRING
// price:DECIMAL
// stock:INTEGER
With aiDescription more accurate 
const Product = sequelize.define("Product", {
  price: {
    type: DataTypes.DECIMAL,
    aiDescription: "Selling price in USD,
      use for price queries",
  },
  stock: {
    type: DataTypes.INTEGER,
    aiDescription: "Units in inventory,
      use for stock checks",
  },
});
// LLM sees full descriptions ✓

Security

Built-in protection

Isolated Sandbox

All LLM-generated code runs inside isolated-vm — a separate V8 context with zero access to your Node.js environment.

Method Whitelist

Only read-only methods are allowed. Blocked methods:

destroy update create drop truncate

Execution Timeout

Queries are automatically killed after 2 seconds. Infinite loops and hanging queries cannot block your process.

Memory Limit

The isolate is capped at 128MB. Memory exhaustion attacks are prevented at the sandbox level.

No Raw SQL

All queries go through Sequelize's model layer. Raw database access and SQL injection are structurally impossible.

Op Translation

Operator strings like "gt" are translated to Op.gt on the host side — never inside the sandbox.
